Stok Sistemi Sadöksan stok mantığı Netsis’e göre özel çalışır. Netsis, irsaliye/fatura kesilmeden ürünü stoktan tam düşmediği için site stoğu doğrudan Netsis stok değildir. Nihai Formül displayStock = netsisStock - netsisPendingQuantity - activeReservedQuantity Alan Açıklama netsisStock Netsis’ten gelen ana stok netsisPendingQuantity Netsis’te bekleyen, henüz irsaliye/faturası kesilmemiş miktar activeReservedQuantity Sitedeki aktif rezervasyon displayStock Sitede gösterilen satılabilir stok Kurallar ☐ netsisPendingQuantity şimdilik manuel tutulabilir. ☐ Netsis API bilgileri henüz alınmadığı için gerçek API entegrasyonu yazılmayacak. ☐ Netsis API gelince netsisPendingQuantity otomatik beslenecek. ☐ Manuel stok çıkışı yapılırken netsisStock - netsisPendingQuantity - activeReservedQuantity değerinin altına düşülmeyecek. ☐ Stok hareketleri append-only log olarak tutulacak. ☐ StockMovement kayıtları update/delete edilmeyecek. ☐ Sayım düzeltmesinde quantity fark/delta olarak tutulacak. Örnek sayım düzeltmesi: Eski stok: 100 Yeni sayım: 85 StockMovement.quantity = -15 StockMovement Tipleri MANUAL_ENTRY MANUAL_EXIT COUNT_ADJUST ORDER_RESERVE ORDER_CANCEL ORDER_REJECT ORDER_FULFILL RETURN_RESTOCK DİKKAT: ORDER_RESERVE ve ORDER_FULFILL fiziksel stok hareketi değildir; stok olay logudur. Fiziksel stoğu MANUAL_ENTRY, MANUAL_EXIT, COUNT_ADJUST ve RETURN_RESTOCK etkiler. 5. Sipariş Akışı B2B siparişler admin onaylı çalışır.
